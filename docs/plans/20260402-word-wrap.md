# Line Wrap Toggle for Diff View

## Overview
Add a `w` key toggle to enable line wrapping in the diff view. Currently, long lines (common in markdown, comments, prose) extend beyond the viewport and require horizontal scrolling. Wrap mode makes these readable without scrolling by breaking lines at word boundaries, showing a `↪` marker on continuation lines. The wrap state is shown as a `↩` icon in the status line mode icons section alongside `▼` (collapsed) and `◉` (filter).

## Context
- **Rendering:** `ui/diffview.go` — `renderDiffLine()` builds each line, `renderDiff()` iterates all lines
- **Collapsed mode:** `ui/collapsed.go` — `renderCollapsedAddLine()`, `renderDeletePlaceholder()` also render lines
- **Horizontal scroll:** `ui/diffview.go` — `scrollX` + `ansi.Cut()` for panning right
- **Cursor math:** `ui/annotate.go` — `cursorViewportY()` counts visual rows for viewport positioning
- **Viewport:** `bubbles/viewport` with `SetContent()`, no built-in soft wrap used
- **ANSI wrapping:** `github.com/charmbracelet/x/ansi` v0.11.6 provides `ansi.Wrap(s, limit, breakpoints)` — already a dependency, no new deps needed
- **Mode icons:** `statusModeIcons()` builds icons from booleans (`collapsed.enabled`, `tree.filter`) — wrap follows the same pattern

## Solution Overview

### Behavior
- `w` key toggles wrap mode on/off (off by default)
- When on: long lines word-wrap at `diffContentWidth()`, continuation lines show `↪` gutter marker
- When on: horizontal scroll (`left`/`right` keys) is disabled, `scrollX` reset to 0
- When off: current behavior unchanged (horizontal scroll works)
- Wrap mode persists across file changes within a session
- Wrap mode is also available as a CLI flag (`--wrap`) and config file option
- Status line mode icons section shows `↩` when wrap is active
- Help overlay updated with `w` key under "View" section

### Visual appearance
```
 + This is a very long line that needs to be
 ↪ wrapped at the viewport boundary so the
 ↪ user can read it without scrolling
```
- Continuation lines use same background style as original line (add/remove/context)
- `↪` marker rendered in muted color, same position as `+`/`-` prefix
- Cursor indicator `▶` only appears on the first visual line of a wrapped diff line

## Technical Details

### Wrapping approach
- `wrapContent(content string, width int) []string` wraps raw text content (without gutter prefix) using `ansi.Wrap`
- Wrap width = `diffContentWidth() - gutterWidth` (gutter = cursor col + change type col = 4 chars: `" + "` or `" ↪ "`)
- Split `ansi.Wrap` result on `\n` to get visual lines
- Caller (renderDiffLine) prepends gutter to each line: first line gets normal prefix (`+`/`-`/space), subsequent lines get `↪` prefix
- Verify `↪` (U+21AA) column width via `ansi.StringWidth` — if 2-wide, adjust gutter spacing
- Each continuation line is a separate `\n`-terminated row in the builder output
- `wrapContent` is the shared method used by both rendering and `wrappedLineCount()` to stay in sync

### Cursor/viewport coordinate impact
- `cursorViewportY()` must count continuation lines from wrapped content above the cursor
- Add a helper `wrappedLineCount(idx int) int` that returns how many visual rows a diff line occupies (1 when not wrapping)
- `cursorViewportY()` sums `wrappedLineCount(i)` instead of counting 1 per line

### Horizontal scroll interaction
- When `wrapMode` is true: skip `ansi.Cut` in render, ignore `left`/`right` scroll keys
- When toggling wrap on: reset `scrollX = 0`
- When toggling wrap off: `scrollX` stays at 0 (user scrolls manually if needed)

### Collapsed mode
- Wrapping applies in collapsed mode too — `renderCollapsedAddLine()` and context lines in `renderCollapsedDiff()` use the same wrapping logic
- Delete placeholders (`renderDeletePlaceholder()`) are short and unlikely to need wrapping, but should support it for consistency

### CLI flag and config
- Add `--wrap` boolean flag (default false)
- Config file: `wrap = true/false`
- Env var: add `env:"REVDIFF_WRAP"` tag explicitly (matching existing flag pattern)
