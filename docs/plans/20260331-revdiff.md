# revdiff — TUI Git Diff Reviewer with Annotations

## Overview
- Terminal UI for reviewing git diffs with inline annotations, built with Go + bubbletea
- Two-pane layout: changed file tree (left, grouped by directory) + colorized diff (right)
- Users add annotations on specific diff lines, output goes to stdout for Claude Code consumption
- Replaces the editor-based loop in the git-review skill with a proper TUI experience

- Diff rendering: show full file with changes highlighted inline (green for additions, red for removals), using `git diff` to identify changed lines
- Annotation UX: inline text input below the annotated line
- Output format: structured text to stdout (file:line + comment)
- Project location: `~/dev.umputun/revdiff`

## Solution Overview
- bubbletea app with two main panes managed by lipgloss layout
- Left pane: tree view of changed files grouped by directory, `*` markers for annotated files, `Tab` to filter
- Right pane: bubbles viewport showing self-rendered colorized diff output
- Annotation layer: parse unified diff line numbers to map viewport lines → file lines, inject annotation lines into viewport content
- CLI: `revdiff [ref] [--staged]`, defaults to uncommitted changes
- On quit: print all annotations to stdout in structured format

## Technical Details

### Diff data pipeline
1. `git diff [ref] --name-only --no-renames` → file list for left pane
2. On file select: `git diff [ref] --no-color --no-ext-diff --unified=3 -- <file>` → unified diff output, parse to identify changed line ranges
3. Read full file content (new version for additions/context, old version via `git show [ref]:<file>` for removals)
4. Build full-file view: merge old removals and new additions into a single line sequence with change markers
5. Build `LineMap`: viewport line index → `{file, lineNum, changeType}`
6. Render with lipgloss: green background for additions, red background for removals, normal for unchanged, styled line numbers
7. Display in viewport with annotation lines injected where they exist

### Diff renderer interface
Define `DiffRenderer` interface in `diff/` package to allow mocking in tests:
```go
type DiffRenderer interface {
    ChangedFiles(ref string, staged bool) ([]string, error)
    FileDiff(ref, file string, staged bool) ([]DiffLine, error)
}
```
`DiffLine` holds parsed line info (number, content, change type). `FileDiff` returns the full file as a sequence of lines — unchanged lines, added lines (`+`), and removed lines (`-`) interleaved at their correct positions. Rendering (colorization, line number formatting) is done in the `ui/` package with lipgloss. Concrete implementation shells out to `git diff` and `git show`. Tests use mock with canned fixtures.

### Annotation storage
```go
const (
    ChangeAdd     = "+"
    ChangeRemove  = "-"
    ChangeContext  = " "
)

type Annotation struct {
    File    string
    Line    int
    Type    string // ChangeAdd, ChangeRemove, ChangeContext
    Comment string
}
```
`Store` struct with methods: `Add`, `Delete`, `Get`, `All`, `FormatOutput`. In-memory `map[string][]Annotation` keyed by filename.

### Output format (stdout on quit)
```
## handler.go:43 (+)
use errors.Is() instead of direct comparison

## store.go:18 (-)
don't remove this validation
```

### Key bindings
- `j/k` or `↑/↓`: scroll diff viewport
- `Enter`: select file from list
- `Tab`: toggle all changed files / only annotated files
- `n/p`: next/prev changed file
- `a`: annotate current diff line (inline text input appears below)
- `d`: delete annotation under cursor
- `Esc`: cancel annotation input
- `q`: quit, output annotations to stdout
